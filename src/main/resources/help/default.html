CloudTrail Viewer

CloudTrail Viewer is a desktop application for viewing and analysing CloudTrail event files.

Loading Events

here are four ways you can load CloudTrail log files into the application these are:
  • Load complete event files from the local file system
  • Load complete event files from a S3 bucket
  • Load only API calls that match a specific criteria from the local file system
  • Load only API calls that match a specific criteria from the local file system

Preferences

AWS Accounts

To load files from S3 you have to provide a set of IAM API keys, this is done on the AWS Tab of the Preferences Dialog. You can provide multiple sets of keys if you wish and switch between them when using the S3 FileChooser.

AWS Aliases

If you are logging multiple accounts into the same S3 bucket you need to remember which 12 digit account belongs to which one of your account. To help you you can enter an alias for the account number so when browsing an S3 bucket the alias will be displayed instead of the number.

Security

The security tab allows you to add and remove API calls that you wish to monitor that relate to potential security issues such as a Securiy Group being opened to allows traffic in through unauthorised ports.

Resources

The resources tab allows you to add and removed API calls that related to resources, for example you might want to check when a new EC2 or RDS is created, or when an EC2 was stopped.
